# 学情智能分析与预警系统 — 产品规格

## 1. 目标用户

- **主要用户**：基础教育、职业教育、高等教育教师（班主任、学科教师、教研员）
- **使用场景**：期中/期末/月考后，快速分析班级和学生学业状况
- **技术能力**：不要求编程基础，能操作 Excel 和网页即可

## 2. 真实痛点

1. **数据看不懂**：手上有大量成绩 Excel 表，但人工分析效率低、维度少，难以发现隐藏规律。
2. **问题发现滞后**：等到成绩明显下滑时才意识到问题，错过最佳干预时机。
3. **个性化指导缺依据**：想给不同学生针对性建议，但缺乏数据支撑，只能凭经验。
4. **汇报耗时**：向学校/家长汇报学情时，手动做图表、写分析费时费力。

## 3. 解决方案

一个基于 Web 的学情分析与预警系统：教师上传成绩 Excel，系统自动完成统计分析、薄弱点识别、风险预警，并生成可下载的 AI 分析报告。

## 4. MVP 核心功能（必须做）

| 功能 | 说明 |
|------|------|
| **成绩导入** | 支持 Excel/CSV 导入，自动识别学生、科目、分数结构 |
| **班级仪表盘** | 展示班级均分、标准差、分数段分布、排名变化趋势 |
| **知识点掌握度热力图** | 将成绩映射到知识点维度，可视化展示全班薄弱点 |
| **学生个体画像** | 每个学生的成绩趋势、薄弱知识点、班级排名变化 |
| **学习风险预警** | 自动标记"连续下滑""大幅退步""偏科严重"等风险学生 |
| **AI 分析报告** | 一键生成自然语言分析 report（班级整体+重点关注学生） |
| **报告导出** | 支持 PDF / Word 格式下载，用于汇报和存档 |

## 5. 当前版本明确不做（非目标）

1. **家长端通知**：不涉及向家长推送消息或建立家长账号体系。
2. **学生个人 APP**：学生不直接使用本系统，系统面向教师。
3. **多学校数据对比**：暂不支持跨校数据汇聚和对比分析。
4. **实时课堂数据采集**：不接入课堂互动、签到、在线答题等实时数据。
5. **自动排课/选课**：不学情无关的功能边界。
6. **在线考试系统**：不支持出题、组卷、在线作答功能。
7. **AI 自动出题**：报告生成外的 AI 能力不在范围内。
8. **数据对接教务系统**：暂不支持从学校现有教务系统自动同步数据。

## 6. 竞品与差异点

| 竞品 | 局限 | 本系统差异 |
|------|------|-----------|
| 学校现有成绩管理系统 | 只有成绩录入和查询，无分析能力 | 深度分析+AI报告+预警 |
| Excel 手工分析 | 耗时、维度单一、难以发现规律 | 自动化、多维可视化、AI解读 |
| 通用 BI 工具（如 PowerBI） | 学习成本高，需自行建模 | 教育场景专用，开箱即用 |

## 7. 已确认的问题

| 问题 | 决策 |
|------|------|
| 成绩 Excel 模板 | 已有固定模板 `一年级.xlsx`，系统按此模板解析 |
| 知识点体系 | 教师自行配置，非内置 |
| AI 报告模型 | 使用云端 API（DeepSeek / 豆包 / 智谱等国产模型） |
| 部署方式 | 单机部署，本地运行 |
| 账号体系 | 极简内置账号（保留登录页用于演示完整性，但无需注册/多角色，内置默认教师账号） |

> **关于"账号体系"的说明**：原型中有登录页是为了让演示流程更完整、更像真实产品。实际实现时只需内置一个默认账号（如 `T2024001` / `admin`），教师打开系统输入预设账号即可使用。不需要注册、忘记密码、权限管理等功能。
